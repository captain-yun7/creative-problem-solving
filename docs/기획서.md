# CPS 스캐폴딩 AI 에이전트 - 종합 기획서

## 1. 프로젝트 개요

**프로젝트명:** Pre-service Teacher Thinking Facilitator

**목적:** 대학생 예비교사를 대상으로, 창의적 문제해결(CPS) 과정에서 메타인지를 촉진하는 **질문 기반 AI 스캐폴딩 시스템** 개발

**핵심 원칙:**

- 답변 제공이 아닌 **사고 촉진** (직접 답변 금지)
- 단계 강제 없는 **학습자 자율 진행**
- 대화 맥락 기반 **적응적 질문 생성**

---

## 2. 핵심 로직 흐름

```
[학습자 입력] → Stage 판단 → State 진단 → Metacognition 선택 → 질문 생성 → [에이전트 응답]
```

### 2.1 CPS 3단계 (Stage)

| 단계                                        | 설명                   |
| ------------------------------------------- | ---------------------- |
| **Understanding the Problem** (문제 이해)   | 문제 이해 및 정보 탐색 |
| **Idea Generation** (아이디어 생성)         | 아이디어 확산적 생성   |
| **Planning for Action** (실행 준비)         | 아이디어 선택 및 정교화 |

### 2.2 상태 진단 (State) - 단계별

| 단계         | 상태 그룹                | 세부 상태                                                          |
| ------------ | ------------------------ | ------------------------------------------------------------------ |
| 문제 이해     | 과제/목표 정의 불명확     | Goal unclear, Task misunderstanding, Goal prioritization confusion |
| 문제 이해     | 정보 수집/통합 부족       | Inadequate information collection/analysis/integration             |
| 아이디어 생성 | 창의적·확산적 사고 부족   | Limited thinking, Insufficient ideas, Overly constrained           |
| 아이디어 생성 | 심층적·체계적 사고 부족   | Shallow thinking, Lack of critical/systematic reasoning            |
| 실행 준비     | 심층적·체계적 사고 부족   | (위와 동일)                                                        |
| 실행 준비     | 평가/의사결정 어려움      | Lack of evaluation criteria, Decision uncertainty                  |

### 2.3 메타인지 요소 (Metacognition) - 3가지

| 유형                  | 역할                   | 순환 구조                              |
| --------------------- | ---------------------- | -------------------------------------- |
| **Knowledge** (지식)  | 전략/개념 활성화       | Knowledge → Monitoring → Control 순환  |
| **Monitoring** (점검) | 현재 상태 자기 점검    |                                        |
| **Control** (통제)    | 전략 수정 및 행동 조절 |                                        |

### 2.4 응답 포맷 (고정 4단 구조)

1. **공감** - 학습자 상태에 대한 짧은 공감
2. **진단** - 상태를 한 문장으로 명확히 진단
3. **질문** - 사고 촉진 질문 **1개만** 제시
4. **재수행 유도** - 다시 과제를 수행하도록 유도

**출력 예시:**

> 아이디어가 잘 떠오르지 않으면 막힌 느낌이 들 수 있어요.
> 지금은 아이디어의 다양성이 부족한 상태로 보여요.
> 다른 관점에서 아이디어를 더 떠올려볼 수 있을까요?
> 새로운 아이디어를 3가지 작성해보세요.

## 3. 프롬프트 전략 (Stage × State × Metacognition 매트릭스)

총 **5개 상태 × 3개 메타인지 = 15가지 질문 생성 전략**

### 3.1 문제 이해 단계

#### A. 과제/목표 정의 불명확

| 메타인지     | 전략                                                                                         |
| ------------ | -------------------------------------------------------------------------------------------- |
| Knowledge    | 산출물 평가 기준(새로움, 유용성, 실현 가능성) 활성화 / 문제 재진술, 조건 파악, 핵심 제약 정리 인식 |
| Monitoring   | 과제 목표를 스스로 설명 / 불명확한 조건 특정 / 현재 해석과 과제 요구 일치도 판단                  |
| Control      | 불명확한 조건 확인을 위한 추가 정보·기준 정하기 / 목표 우선순위 결정                             |

#### B. 정보 수집 및 통합 부족

| 메타인지     | 전략                                                                           |
| ------------ | ------------------------------------------------------------------------------ |
| Knowledge    | 필요한 개념/배경지식 활성화 / 정보를 문제와 연결                                |
| Monitoring   | 현재 정보만으로 해결안 가능한지 / 빠진 핵심 정보 / 정보 간 연결성 질문          |
| Control      | 필요 정보 목록 재수립 / 수집 정보 범주화·표 정리 / 해결안 연결 정보만 추리기    |

### 3.2 아이디어 생성 단계

#### C. 창의적·확산적 사고 부족

| 메타인지     | 전략                                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------- |
| Knowledge    | 브레인스토밍, 속성나열, 관점바꾸기, 결합, 제약뒤집기 등 생성 전략 활성화                         |
| Monitoring   | 아이디어 상호 차별성 / 같은 방향 반복 여부 / 추가 생성 여지 질문                                 |
| Control      | 미사용 전략 강제 적용 / 반대입장·다른 맥락 아이디어 / 수+범주 동시 확장 목표 부여                 |

#### D. 심층적·체계적 사고 부족 (아이디어 생성 + 실행 준비 공통)

| 메타인지     | 전략                                                                                   |
| ------------ | -------------------------------------------------------------------------------------- |
| Knowledge    | 유용성, 적합성, 실현 가능성 등 평가 기준 인식 / 구체화 요소(대상, 맥락, 작동, 효과) 활성화 |
| Monitoring   | 왜 효과적이라 생각하는지 / 문제 핵심과 실제 연결 여부 질문                                |
| Control      | 가장 약한 부분 보완 / 실행 맥락 추가 구체화 / 아이디어 결합으로 대안 강화                   |

### 3.3 실행 준비 단계

#### E. 평가 및 의사결정 어려움

| 메타인지     | 전략                                                                                         |
| ------------ | -------------------------------------------------------------------------------------------- |
| Knowledge    | 새로움, 유용성, 실현 가능성, 윤리성 등 평가 기준 명확화 / 기준 근거 판단 인식                   |
| Monitoring   | 평가 기준 일관성 / 모든 아이디어에 동일 적용 여부 / 특정 선호에 의한 왜곡 확인                   |
| Control      | 평가 기준으로 아이디어 비교 / 선택 이유 언어화 / 상위 2개 남기고 재평가                         |

## 4. 화면 구성 (UI)

### 4.1 공통 레이아웃

```
┌──────────────────────────────────────────────────────────────────────┐
│  상단: 단계 진행 표시  [문제 이해] ──→ [아이디어 생성] ──→ [실행 준비]  │
├──────────────┬───────────────────────────┬───────────────────────────┤
│  좌측 사이드바  │      중앙 채팅 영역        │      우측 사이드바         │
│              │                           │                           │
│  ● 단계별     │  AI 에이전트 대화          │  ● 사고 과정 체크리스트    │
│    팁 & 예시  │  (질문/응답)               │    - 현재 상태 자가 진단   │
│              │                           │    - 체크박스 항목          │
│  ● 사고 촉진  │  ┌─────────────────────┐  │                           │
│    질문 버튼  │  │ 메시지 입력창        │  │  ● 자유 메모 영역          │
│  - 확산적 사고 │  └─────────────────────┘  │                           │
│  - 수렴적 사고 │                           │                           │
├──────────────┴───────────────────────────┴───────────────────────────┤
│  하단: 사고 도구 바 (단계별 변경)                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 4.2 단계별 화면 차이

#### 문제 이해 단계

- **사고 도구:** 분석 카드/도구
- **좌측 촉진 질문 (확산적):** "이 과제에서 주목해야 하는 것은?", "어떤 정보를 확인해야 할까요?" 등
- **좌측 촉진 질문 (수렴적):** "최종적인 목표와 관심사항은?", "어떤 자료가 가장 중요하게 고려되어야 할까?" 등
- **우측 체크리스트:**
  - 과제 목표 설정: 목표 이해도, 무엇을 해야 할지, 우선순위 명확성
  - 정보 탐색: 필요 정보 인지, 정보 이해 여부, 정보 연결 가능성

#### 아이디어 생성 단계

- **사고 도구:** 브레인스토밍, 스캠퍼, 강제결합법, 속성기입, 목록사고, 시네틱스
- **좌측 촉진 질문 (확산적):** "어떤 아이디어를 생각해볼 수 있을까요?", "다른 새롭고 독특한 방법은?" 등
- **좌측 촉진 질문 (수렴적):** "어떤 아이디어가 가장 흥미롭거나 가능성이 있어 보이나요?" 등
- **우측 체크리스트:**
  - 확산적 사고: 아이디어 충분성, 다양한 관점, 참신성
  - 비판적 사고: 현실적이고 논리적인지, 근거 충분성

#### 실행 준비 단계

- **사고 도구:** 하이라이팅, 히트 기입, PMI, 평가 행렬표, 역브레인스토밍
- **좌측 촉진 질문 (확산적):** "제시된 아이디어에 어떤 기준을 적용할 수 있을까요?" 등
- **좌측 촉진 질문 (수렴적):** "어떤 아이디어가 최우선 순위에 올라야 할까요?" 등
- **우측 체크리스트:**
  - 평가 및 의사결정: 아이디어 판단 용이성, 선택 확신, 기준 기반 결정

└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│           Service Layer                      │
│  - GeminiService (LLM Integration)          │
│  - Prompt Engineering Logic                 │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│        Business Logic Layer                  │
│  - CPS Stage Detection                      │

                  │

## 6. 핵심 설계 규칙

### 6.1 응답 깊이 평가 기준

| 깊이    | 기준                                      | 후속 처리                |
| ------- | ----------------------------------------- | ------------------------ |
| Shallow | 50자 이하, 구체성 부족, 단일 관점          | 동일 메타인지 요소로 추가 탐색 |
| Medium  | 50-150자, 일부 구체적, 2-3개 아이디어      | 다음 메타인지 요소로 진행  |
| Deep    | 150자 이상, 다양한 관점, 구체적 예시       | Deep 2회 이상 시 단계 전환 고려 |

### 6.2 단계 전환 우선순위

1. 사용자 명시적 요청 (키워드: "아이디어", "다음 단계", "실행 준비")
2. Gemini의 `should_transition=true` 권고
3. 최소 요구사항 충족 (예: 원인 3개 이상, 아이디어 5개 이상)

### 6.3 전환 제약

- 역방향 전환 불가 (문제 이해 ← 아이디어 생성)
- 순차적 진행 권장 (건너뛰기 방지)
- 턴 제한 도달 시 자동 전환 없음 (알림만 제공)

### 6.4 단계별 메타인지 순환 패턴

| 단계           | 순환 패턴                        |
| -------------- | -------------------------------- |
| 문제 이해       | Knowledge → Monitoring → Control (반복) |
| 아이디어 생성   | Monitoring → Control (반복)      |
| 실행 준비       | Monitoring → Control → Knowledge (순차) |

## 7. API 엔드포인트

### Chat API (`/api/chat`)

| Method | Path       | 설명                        | Request Body                              |
| ------ | ---------- | --------------------------- | ----------------------------------------- |
| POST   | `/session` | 새 세션 생성                | `{ "assignment_text": "과제 텍스트" }`     |
| POST   | `/message` | 메시지 전송 + 스캐폴딩 응답 | `{ "session_id", "message", "conversation_history", "current_stage" }` |

### Research API (`/api/research`)

| Method | Path                                  | 설명              |
| ------ | ------------------------------------- | ----------------- |
| GET    | `/sessions`                           | 모든 세션 조회    |
| GET    | `/sessions/{session_id}/conversations`| 대화 히스토리     |
| GET    | `/sessions/{session_id}/metrics`      | 세션 지표         |
| GET    | `/export/conversations/csv`           | 대화 CSV Export   |
| GET    | `/export/metrics/csv`                 | 지표 CSV Export   |

## 10. 보안 및 연구 윤리

- **개인정보 최소화:** user_id 선택사항, 이메일/이름/IP 수집 안 함
- **DB 보안:** PostgreSQL SSL, 환경 변수 관리 (.env)
- **CORS:** Production에서 특정 도메인만 허용
- **연구 동의:** 세션 생성 시 데이터 수집 동의 체크박스
- **데이터 보관:** 연구 종료 후 6개월, 이후 자동 삭제
