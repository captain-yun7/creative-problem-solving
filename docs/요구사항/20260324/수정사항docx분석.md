# 수정사항 분석 (2026.03.24)

## 수정사항 요약

### 1. 응답 생성 포맷 수정 — 힌트 추가 + 질문 구조화

기존 4단 포맷에서 **5단 포맷**으로 변경:

| 순서 | 항목 | 내용 | 변경 여부 |
|------|------|------|-----------|
| 1 | **공감 (Empathy)** | 학습자의 현재 상태에 대한 짧은 공감 | 유지 |
| 2 | **진단 (Diagnosis)** | 학습자의 상태를 한 문장으로 명확히 진단 | 유지 |
| 3 | **질문 (Question)** | 창의적 메타인지 촉진 질문 1개 | **수정** |
| 4 | **힌트 (Hint)** | 방향 제시용 키워드 2~3개 | **신규 추가** |
| 5 | **재수행 유도 (Action Prompt)** | 행동 단위를 명시 | **수정** |

#### 질문(Question) 변경 사항
- **명확한 기준 제시** 필요: "무엇을 기준으로 / 어떤 방식으로 / 어떻게 바꿔서 생각할지"
- **인지적 피로도 방지**: 이미 수행한 사고를 반복 요구하지 않기. 충분한 수행이 이루어진 경우 인정하고 다음 단계로 안내

#### 힌트(Hint) 신규 추가
- 학습자의 과제 수행을 돕는 **방향 제시용 키워드 2~3개** 제시

#### 재수행 유도(Action Prompt) 변경 사항
- 행동 단위를 명시 (예: "위 질문과 힌트를 참고하여 새로운 아이디어를 2가지 이상 작성해보세요.")

---

### 2. 출력 가독성 개선

- **줄바꿈**을 활용하여 사용자 입장에서 가독성 있게 제시
- 현재 응답이 한 덩어리로 보이는 문제 해결 필요

---

### 3. 대화 턴 알림 제거

- 이전 에이전트 코드에서 참고된 **대화 턴 알림이 뜨지 않도록** 수정
- 학습자에게 "남은 턴: X회" 같은 표시가 노출되지 않아야 함

---

### 4. 설계방향 문서 반영 확인

- 이전에 전달된 **'설계방향 Docx 파일'의 전략과 예시**가 코드(System Prompt)에 충분히 반영되었는지 재확인 요청
